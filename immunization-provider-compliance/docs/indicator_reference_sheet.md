# Indicator reference sheet

The formal indicator matrix is stored at `data/reference/indicator_matrix.csv`. This page explains how to read the M&E outputs.

## Target status labels

- **On track**: the indicator meets the target threshold.
- **Needs attention**: the indicator misses the target threshold.
- **Review denominator**: the denominator is small or missing and the indicator should not be interpreted without checking counts.

## Key M&E files

| File | Purpose |
|---|---|
| `indicator_performance_by_quarter.csv` | Quarterly monitoring dataset with all core indicators. |
| `target_tracking_summary.csv` | Latest-period and all-period target status for each indicator. |
| `ta_coverage_by_provider_risk.csv` | Technical assistance coverage by risk tier. |
| `followup_timeliness_summary.csv` | Observed follow-up timing after non-compliant visits. |
| `pre_post_evaluation_summary.csv` | Pre/post comparison around the simulated 2022 TA-targeting period. |
| `quarterly_program_brief.md` | Plain-language program brief generated from the current outputs. |

## Practical note

The targets are synthetic and are included to demonstrate M&E workflow design. In a real program, targets should be agreed with program leadership and based on policy, baseline performance, feasibility, and field capacity.
