# Logic model: immunization provider compliance monitoring

This synthetic project treats provider compliance monitoring as a public health program, not only as a statistical dataset.
The logic model below describes the pathway from routine program inputs to expected short- and medium-term outcomes.

| Logic model level | Project-specific content |
|---|---|
| Inputs | Provider registry, site visit forms, immunization program staff, technical assistance tools, data systems, analytic staff, dashboard/reporting process. |
| Activities | Schedule provider site visits, review compliance domains, provide technical assistance, document findings, clean program data, monitor trends, identify high-risk providers. |
| Outputs | Completed visits, providers receiving technical assistance, quarterly indicator reports, high-risk provider lists, data-quality checks, dashboard-ready exports. |
| Short-term outcomes | Improved completeness of compliance documentation, faster follow-up, better targeting of technical assistance, more consistent reporting across jurisdictions. |
| Intermediate outcomes | Lower provider non-compliance, fewer repeat non-compliance findings, improved storage/handling and inventory practices, stronger routine program monitoring. |
| Long-term public health contribution | Stronger immunization program quality and reduced operational risk in vaccine management. |

The project does not claim that technical assistance caused improvement. It evaluates whether the monitoring data are consistent with better targeting and whether program indicators are moving in the expected direction.
