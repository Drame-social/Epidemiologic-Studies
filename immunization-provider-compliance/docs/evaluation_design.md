# Evaluation design

## Evaluation type

This is a mixed monitoring and evaluation design using synthetic routine program data. It combines descriptive monitoring, pre/post comparison, targeted technical assistance coverage assessment, and regression-based outcome modeling.

## Program period

The project covers synthetic site visits from 2019 through 2025. The year 2022 is treated as the start of a simulated technical-assistance targeting period.

## Primary evaluation comparison

The evaluation compares 2019-2021 with 2022-2025. This is not a causal design because providers were not randomized and the synthetic intervention is represented only through program variables and time periods.

## Outcomes

Primary outcome:
- Overall non-compliance among analysis-ready visits.

Secondary outcomes:
- Domain-specific non-compliance, especially documentation, inventory, and storage/handling.
- Compliance score, defined as the count of compliant domains from 0 to 5.
- Technical assistance coverage among high-risk visits and providers.
- Observed follow-up within 365 days after a non-compliant visit.

## High-risk definition

A visit is considered high risk if it has at least two of the following flags: prior non-compliance, staff turnover, rural location, private clinic provider type, site-visit delay greater than 30 days, or current non-compliance. This is a pragmatic monitoring definition, not a validated risk score.

## Analytic approach

- Quarterly indicator trend monitoring.
- Target tracking against a predefined indicator matrix.
- Pre/post comparison with risk difference and relative change.
- Stratified monitoring by provider type and rural/urban status.
- Logistic regression for adjusted association with non-compliance.
- Survival-style analysis for time to first observed non-compliance.

## Interpretation standard

The evaluation emphasizes operational decision-making. A statistically significant result is not automatically treated as programmatically important, and a missed target is not treated as failure without reviewing data quality, denominator size, and field context.
