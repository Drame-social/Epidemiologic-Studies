# Bias and Limitations
1. **Synthetic data.** Associations are those built into the generator; nothing here is a real estimate. 2. **Single-source generation** simplifies confounding and measurement error relative to real cohorts. 3. **Complete-case modeling.** Regression and Cox drop rows missing any model variable; with real data, characterize the missingness mechanism and consider imputation. 4. **Proportional hazards.** The Cox model assumes proportional hazards; test it (Schoenfeld residuals) and consider time-varying effects in real work. 5. **Time-series seasonality** is synthetically induced through seasonal event ascertainment; the forecast shows method mechanics, not a real signal. 6. **Cross-language parity.** Python is executed and validated here; R and SAS are complete and produce the same key tables and dashboard exports, but require a local R or SAS installation to populate output folders. Run  or submit  after setting  in . SAS requires manual path configuration before execution.
7. **Time-series stationarity.** The ADF test on monthly MACE counts returns p = 0.369, failing to reject the unit-root null hypothesis; the series is not stationary at conventional thresholds. The additive Holt-Winters forecast and seasonal decomposition remain valid as descriptive and exploratory methods in this context, but inference from the trend component should be treated cautiously. In real data, differencing or an ARIMA-family model would be appropriate before forecasting a non-stationary series.
